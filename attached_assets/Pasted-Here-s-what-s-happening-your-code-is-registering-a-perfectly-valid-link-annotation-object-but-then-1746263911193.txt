Here’s what’s happening: your code is registering a perfectly valid link annotation object, but then you’re clobbering whatever annotations already exist on that page by doing

currentPage.node.set(
  PDFName.of('Annots'),
  pdfDoc.context.obj([linkAnnotation])
);

every time you add one. Acrobat Reader then only “sees” the last annotation—or none if something got dropped—so nothing ever ends up clickable  ￼.

Key points
	1.	Register the annotation correctly
Use pdfDoc.context.register() to get a PDFObjectReference, with keys as PDFName and the URI as a PDFString. For a URI link annotation you need at minimum:

const linkAnnotationRef = pdfDoc.context.register(
  pdfDoc.context.obj({
    Type: 'Annot',  
    Subtype: 'Link',  
    Rect: [x1, y1, x2, y2],  
    Border: [0, 0, 0],  
    A: {  
      S: 'URI',  
      URI: yourUrlString  
    }
  })
);

This matches the PDF spec for a hyperlink annotation  ￼.

	2.	Append to the existing /Annots array
Instead of overwriting, retrieve any existing annotations array and push your new one onto it. For example:

import { PDFArray, PDFName } from 'pdf-lib';

// …after registering linkAnnotationRef…

// Try to grab existing Annots
let annots = currentPage.node.lookup(PDFName.of('Annots'), PDFArray);

if (annots) {
  // Push onto it
  annots.push(linkAnnotationRef);
} else {
  // No annotations yet: create a new array
  currentPage.node.set(
    PDFName.of('Annots'),
    pdfDoc.context.obj([linkAnnotationRef])
  );
}

This way, every annotation you add remains in the page’s /Annots array  ￼.

	3.	(Optional) Add a visible border for debugging
If you still can’t see or click the region, give the annotation a border and color so you can confirm it’s there:

Border: [0, 0, 2],   // 2-pt border
C: [0, 0, 1],        // blue border

You’ll then see exactly where Acrobat thinks the link lives  ￼.

	4.	Check Acrobat Reader’s settings
In Acrobat Reader DC, go to Edit → Preferences → General and ensure “Create links from URLs” is enabled. Otherwise Reader may ignore link annotations  ￼.

Why this fixes it
	•	Correct annotation dictionary: Specifying /S(/URI) and /URI ensures the PDF reader knows it’s a web link action  ￼.
	•	Appending to /Annots: Prevents your code from wiping out earlier annotations, so every link you add is preserved  ￼.
	•	Visible border: Helps verify the annotation rect is in the right place  ￼.
	•	Reader prefs: Some PDF viewers won’t honor links unless that option is on  ￼.

Once you make those changes, open the PDF in Acrobat Reader—and you should finally be able to click your “Play Voice Message” links.
